# Demo Script

## Setup

```bash
cp .env.example .env
# Add GETLARK_API_KEY, GITHUB_TOKEN, and optionally KIRO_API_KEY.
docker compose up --build
```

Open:

- `http://localhost:3000`
- `http://localhost:7681`

## Story Beats

1. Show the browser shell readiness panel.
2. Open the terminal desk and search the cold-case database.
3. Select any case to prove the system is not hard-coded.
4. For the video, use `Maven generate-sources cannot resolve a dependency` because it is likely to produce crisp build evidence.
5. Create a run and show the generated workspace.
6. Start the pipeline.
7. Show Kiro reconstruction creating the failing reproduction and stopping before repair.
8. Show Lark reproduction evidence as the forensic handoff.
9. Show Kiro repair from `prompts/kiro-repair-from-lark.md`.
10. Show Lark final verification and the `Closure Granted` verdict.
11. Show the creative case file and Case Closed Gallery.

## Pitch Line

Old unanswered Stack Overflow questions are unresolved support incidents. Coding Cold Cases Cracker turns each one into a reproducible lab case, then uses Lark as the independent evidence gate before calling it closed.
